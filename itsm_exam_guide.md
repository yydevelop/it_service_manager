# ITサービスマネージャ試験 完全ガイド

## 1. ITサービスマネージャ試験の概要

### 1.1 試験の位置づけ

ITサービスマネージャ試験は、情報処理技術者試験における高度試験の一つであり、**ITサービスの安定稼働や継続的改善を担う専門家**を認定する国家資格です。

### 1.2 対象者像

- **ITサービスの運用・保守に従事するマネージャー**
- サービス提供における品質管理や改善活動を主導する立場の方
- ITサービスの企画、設計、移行、運用のライフサイクル全般を統括する役割

### 1.3 試験の難易度

- **合格率：約14～15%**（令和元年～令和5年度の平均）
- 高度試験の中では比較的合格率が高い区分
- ただし、午後Ⅱ（論文）対策が重要

## 2. 試験の構成と出題範囲

### 2.1 試験時間と形式

| 試験区分 | 試験時間 | 出題形式 | 合格基準 |
|---------|---------|---------|---------|
| 午前Ⅰ | 9:30～10:20（50分） | 多肢選択式（四肢択一）30問 | 60点以上/100点 |
| 午前Ⅱ | 10:50～11:30（40分） | 多肢選択式（四肢択一）25問 | 60点以上/100点 |
| 午後Ⅰ | 12:30～14:00（90分） | 記述式 3問中2問選択 | 60点以上/100点 |
| 午後Ⅱ | 14:30～16:30（120分） | 論述式 2問中1問選択 | ランクA |

### 2.2 午前Ⅰの出題範囲

**他の高度試験と共通**

- テクノロジ系（約50%）
  - 基礎理論、コンピュータシステム、技術要素
- マネジメント系（約30%）
  - プロジェクトマネジメント、サービスマネジメント
- ストラテジ系（約20%）
  - システム戦略、経営戦略、企業と法務

**免除制度**
- 応用情報技術者試験合格者は2年間免除
- 他の高度試験合格者も2年間免除

### 2.3 午前Ⅱの出題範囲

**ITサービスマネージャ試験固有の専門知識**

主要出題分野：
1. **サービスマネジメント（ITIL準拠）**
   - サービス戦略、サービス設計
   - サービス移行、サービス運用
   - 継続的サービス改善（CSI）

2. **システム監査**
   - 監査の目的、計画、実施
   - 監査証跡、監査報告

3. **サービスマネジメントプロセス**
   - インシデント管理
   - 問題管理
   - 変更管理
   - リリース管理
   - 構成管理
   - サービスレベル管理
   - キャパシティ管理
   - 可用性管理
   - ITサービス継続性管理
   - 情報セキュリティ管理

### 2.4 午後Ⅰの出題傾向

**長文読解型の記述式問題**

- 問題文：A4で6～8ページ程度
- 解答形式：25～50字程度の記述が中心
- 出題テーマ例：
  - サービスレベル管理とSLA
  - インシデント管理・問題管理
  - 変更管理・リリース管理
  - ITサービス継続性管理（BCP/DR）
  - キャパシティ管理・性能管理
  - サービスデスク運営
  - 外部委託管理

**解答のポイント**
- 問題文中の記述を根拠に論理的に解答
- ITサービスマネージャの視点（管理者目線）
- 具体的な数値、事実（ファクト）に基づく判断

### 2.5 午後Ⅱの出題傾向

**論述式（小論文）**

設問構成：
- **設問ア（800字以内）**：ITサービスの概要と問題点
- **設問イ（800～1,600字）**：問題に対する対策・工夫
- **設問ウ（600～1,200字）**：評価と今後の改善

出題テーマ例：
- ITサービスの品質管理
- サービスレベルの維持・向上
- インシデント対応と再発防止
- 変更管理・移行管理
- ITサービス継続性の確保
- 運用の効率化・自動化
- 外部委託先の管理
- 利用者教育とサービスデスク

## 3. 各試験の対策方法

### 3.1 午前Ⅰ対策

**基本戦略**
- 免除制度の活用を検討
- 過去問の繰り返し学習が効果的
- 高度試験共通問題のため、他区分の過去問も活用可能

**学習方法**
1. 過去問題集を3年分以上解く
2. 間違えた問題は解説を熟読
3. 類似問題を解いて定着確認
4. 午前Ⅰ免除制度を活用できる場合は積極的に利用

### 3.2 午前Ⅱ対策

**基本戦略**
- **ITIL（IT Infrastructure Library）の理解が必須**
- サービスマネジメントプロセスの体系的理解
- 過去問の反復が最も効果的

**重点学習分野**
1. ITILのサービスライフサイクル
   - サービス戦略
   - サービス設計
   - サービス移行
   - サービス運用
   - 継続的サービス改善

2. 各種管理プロセス
   - インシデント管理
   - 問題管理
   - 変更管理
   - リリース管理
   - 構成管理（CMDB）
   - サービスレベル管理（SLA/SLO）

3. 品質管理・性能管理
   - KPI/KGI
   - PDCA/PDSA
   - MTBF/MTTR/MTRS

**学習スケジュール**
- 試験3ヶ月前：ITIL基礎知識の習得
- 試験2ヶ月前：過去問演習開始（5年分）
- 試験1ヶ月前：弱点分野の集中学習

### 3.3 午後Ⅰ対策

**基本戦略**
- 問題文の読解力が重要
- **問題文中から根拠を見つける能力**
- ITサービスマネージャの視点（管理者目線）での思考

**解答テクニック**
1. **問題文の構造を把握**
   - 登場人物、組織構成
   - システム構成、サービス内容
   - 課題・問題点の所在

2. **設問要求の正確な理解**
   - 「誰の立場で」答えるか
   - 「何を」問われているか
   - 字数制限の確認

3. **解答の構成**
   - 結論を先に述べる
   - 理由・根拠を明確に
   - 問題文の表現を活用

**時間配分**
- 問題選択：5分
- 問題文読解：20分
- 解答作成：60分
- 見直し：5分

**過去問演習**
- 最低でも過去3年分（9問）を解く
- 解答例と講評を必ず確認
- 自分の解答と比較し、改善点を明確化

### 3.4 午後Ⅱ対策

**最重要ポイント**
- **事前準備が合否を分ける**
- 自身の経験に基づく事例の準備
- 論文骨子（テンプレート）の作成

**論文作成の基本方針**

1. **設問アの記述内容**
   - ITサービスの概要
     - 対象サービスの特性
     - 合意しているSLA
     - 組織体制、役割
   - 存在する問題点
     - 具体的な事象（数値で示す）
     - 問題の影響範囲
     - 背景・原因

2. **設問イの記述内容**
   - 問題に対する対策
     - 実施した方策
     - ITサービスマネージャとしての工夫
     - 具体的な実施内容
   - 管理者視点の創意工夫
     - プロセス改善
     - 予兆検知
     - 根本原因分析

3. **設問ウの記述内容**
   - 評価
     - 定量的な効果測定（KPI）
     - 達成度の確認
   - 今後の改善策
     - 残存リスクへの対応
     - 継続的改善（CSI）の視点

**論文作成の注意点**

❌ **NGパターン**
- 担当者目線の記述（管理者視点が不足）
- ITサービスではなく情報システムの概要を書く
- 抽象的な記述（具体性の欠如）
- 定量的評価がない
- 1つの視点のみ（複数視点が必要）

⭕ **OKパターン**
- ITサービスマネージャの視点
- 顧客志向の高いレベルでのマネジメント
- 具体的な数値、ファクトに基づく判断
- 複数の評価軸での分析
- 継続的改善（CSI）の姿勢

**事例準備のポイント**

準備すべきテーマ（最低3つ）：
1. サービスレベル管理（SLA/SLO）
2. インシデント・問題管理
3. 変更管理・移行管理
4. ITサービス継続性管理
5. サービスデスク運営改善

各テーマで準備する内容：
- サービス概要（300字）
- 問題点（300字）
- 対策・工夫（800字）
- 評価・改善（500字）

**論文骨子の作成**

```
【設問ア：ITサービスの概要と問題点】
1. 企業・事業の概要（100字）
2. ITサービスの概要（200字）
   - サービス内容
   - SLA
   - 組織体制
3. 問題点（500字）
   - 具体的事象（定量的に）
   - 影響範囲
   - 原因分析

【設問イ:対策と工夫】
1. 実施した方策の概要（200字）
2. ITサービスマネージャとしての工夫（600字）
   - 具体的施策①
   - 具体的施策②
   - 関係者調整
3. 実施における留意点（400字）

【設問ウ：評価と改善】
1. 評価（400字）
   - 定量的効果（KPI）
   - 達成度
2. 今後の改善策（400字）
   - 残存課題
   - 継続的改善の方針
```

**時間配分**
- 問題選択：5分
- 構成検討：10分
- 設問ア：30分
- 設問イ：50分
- 設問ウ：20分
- 見直し：5分

**過去問演習**
- 過去5年分の問題・解答例・講評を確認
- 良い論文の特徴を分析
- 自分で論文を書いて時間感覚を養う

## 4. 学習スケジュール（試験6ヶ月前からの計画）

### 4.1 試験6ヶ月前～4ヶ月前

**目標：基礎知識の習得**

- ITILの学習（参考書1冊を通読）
- サービスマネジメントプロセスの理解
- 午前Ⅱ過去問開始（1年分）

### 4.2 試験4ヶ月前～2ヶ月前

**目標：知識の定着と午後Ⅰ対策**

- 午前Ⅱ過去問演習（3～5年分）
- 午後Ⅰ過去問演習開始（3年分）
- 論文テーマの選定と事例収集

### 4.3 試験2ヶ月前～1ヶ月前

**目標：午後Ⅱ準備と弱点補強**

- 午後Ⅰ過去問演習（追加3年分）
- 午後Ⅱ論文骨子作成
- 事例の肉付け（3テーマ）
- 弱点分野の集中学習

### 4.4 試験1ヶ月前～試験直前

**目標：総仕上げと本番対策**

- 午前Ⅱ総復習
- 午後Ⅰ時間配分の確認
- 午後Ⅱ論文の手書き練習（最低2本）
- 模擬試験（時間を計測）

## 8. まとめ

ITサービスマネージャ試験は、**実務経験とITIL知識の融合**が求められる試験です。

**合格への3つのカギ**

1. **ITILの体系的理解**（午前Ⅱ・午後Ⅰ）
2. **問題文読解力**（午後Ⅰ）
3. **事前準備と論文力**（午後Ⅱ）

特に午後Ⅱ（論文）は、**事前準備の質と量が合否を分けます**。自身の経験を基にした具体的な事例を複数準備し、ITサービスマネージャの視点で論述できるよう訓練しましょう。

計画的な学習と過去問演習を重ねることで、合格は十分に可能です。頑張ってください!
